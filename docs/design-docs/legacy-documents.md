# Legacy Documents

## 概述

本文件说明如何使用 `legacy-*` 文档。它们保留设计演进记录，但不是当前实现的权威依据。

## 核心要点

- legacy 文档的价值在于解释“为什么收敛成现在这样”。
- 当前实现应优先读取：
  - [core-beliefs.md](./core-beliefs.md)
  - [backend-spec.md](./backend-spec.md)
  - [api-contract.md](./api-contract.md)
- 产品愿景母本已迁移到 [../product-specs/product-vision.md](../product-specs/product-vision.md)。

## 约束

- 不要直接把 legacy 文档当成当前契约使用。
- 若某个历史方案要重新启用，必须先补新的设计文档或执行计划。
